## 2. Product Detail Page (PDP)

### 2a. Dynamic OG Image

**What it shows:** 1200×630 branded card — product photo on left, product name + price + "The Possah / Haute Couture" on dark green (`#1F3A2D`) right panel.

**Generated by:** `app/(shop)/[gender]/[category]/[slug]/opengraph-image.tsx`

**How it works:** Next.js calls this at request time, fetches the product's first image and price from Supabase via REST, and renders a PNG using `next/og` (`ImageResponse`).

> **Important:** Do NOT use the Supabase JS client (`createPublicClient`) inside this file — it crashes the OG image sandbox. Always use direct `fetch` to the Supabase REST API. The `formatINR()` helper uses a manual regex instead of `toLocaleString('en-IN')` for the same reason (locale not available in the OG runtime).

**To change layout or colours:** Edit `opengraph-image.tsx` directly. The dark green is `#1F3A2D`, cream is `#F4ECDF`, gold is `#C8973A`.

**To change which product image is used:** It always picks the image with the lowest `position` value in `product_images`. Update that in Supabase → product_images.

---

### 2b. Product Schema (JSON-LD)

Enables Google Shopping rich results — product name, price, availability, brand, images.

**File:** `app/(shop)/[gender]/[category]/[slug]/page.tsx` → `productSchema` object (~line 229)

**What's included:** name, description, all product images, SKU (product UUID), brand, offer price + currency, InStock/OutOfStock, aggregate rating (if reviews exist).

---

### 2c. BreadcrumbList Schema

Breadcrumb trail shown in Google results below the page title.

**File:** `app/(shop)/[gender]/[category]/[slug]/page.tsx` → inside the returned JSX

---

### 2d. Twitter / OG Meta Tags

Both og: and twitter: tags are set per-product using the product's first image.

| Tag | Value |

| og:type | website |
| og:image | First product CDN image (1200×630) |

**File:** `generateMetadata()` in `app/(shop)/[gender]/[category]/[slug]/page.tsx`

## 3. Category Pages

### 3a. OG Image

**Status:** Code is live — emits `og:image` **only if** `category.image_url` is set in Supabase.

**Action needed:** For every category that should show a branded image when shared on social, upload a 1200×630 image to the CDN and paste the URL into the `image_url` column of that category row in Supabase → categories.

**Currently empty:** Sarees (and likely all other categories) — they show the fallback hero placeholder, so no OG image is emitted.

**File:** `app/(shop)/[gender]/[category]/page.tsx` → `generateMetadata()` → `openGraph.images`

## 5. Journal Articles

### 5a. OG Image

**Source:** `featured_image` column in `journal_articles` Supabase table.

**Action needed:** Every article should have a `featured_image` set. Without it, no OG image is emitted and social shares show the homepage default instead.

**File:** `app/(shop)/journal/[slug]/page.tsx` → `generateMetadata()`

---

### 5b. NewsArticle Schema (JSON-LD)

Enables Google Top Stories / AMP eligibility for journal articles.

**File:** `app/(shop)/journal/[slug]/page.tsx` → `<script type="application/ld+json">` inside the page component

**Fields populated automatically:** headline (article title), datePublished, image (featured_image if set), author (Organization: The Possah), publisher logo.

**Publisher logo — current URL:**
```
https://cdn.thepossah.com/1781254169345-name.webp
```

**To update the logo:** Upload the new image to CDN, then change the URL in the script block → `publisher.logo.url`.

**Google's logo spec:** PNG or WebP, opaque background, min 60×60 px, recommended 600×60 px (wide format).

## 6. Share Drawer (PDP)

- **Desktop / unsupported:** opens a popover with Copy Link, WhatsApp, Pinterest

**File:** `components/pdp/ShareDrawer.tsx`

## Quick Reference — Where to Change Each Image

|------|---------------------|---------------|
| Homepage default OG image | `public/images/og-default.jpg` | Upload replacement → update URL in `app/layout.tsx` |
